In the era of agentic AI, **documentation is code**. While a C# file encodes algorithms and domain rules into machine-level data structures (IL, JIT instructions, metadata), documentation provides the necessary "meta-layer."

### Documentation as Ontology

In ontological terms, data, information, and documentation are layers of a single formal representation. Data and code are the raw symbols and executable logic, while documentation explains the structure and the intended use—in other words, the "why."

Documentation is ontology itself: the meaning of classes, design decisions, constraints, and usage guidelines are ensuring the same structure is applied correctly across different systems and domains.

### Prompts Need Structure, Not Just Words

A prompt could be that kind of unsettling raw facts with no context or interpretation—that's why you need to input documentation, not mere words.

Documentation **describes data and processes so they can be understood, used, and reused correctly**.

### Garbage In, Garbage Out

The "Garbage In, Garbage Out" (GIGO) mechanism is the primary reason AI projects fail in production after succeeding in demos. A prompt without structure risks producing errors that are difficult to evaluate without manual correction.

### Why Format Matters

Standard document formats like PDF/DOCX introduce noise that degrades LLM performance. Extraction can result in data loss, tables become unreadable, and OCR errors introduce further noise.

Because LLMs were trained primarily on structured text, TXT, MD, and XML are natively understood best.

**Markdown** is human-readable and provides structure. **Mermaid** is the de facto standard for flowcharts and complex UI structures—it's textual, AI-ready, easy to generate or modify, and understandable without rendering.
