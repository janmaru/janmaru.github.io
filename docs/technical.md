# Technical Analysis

## Build System

- **Platform:** GitHub Pages (static hosting, automated Jekyll builds on push)
- **Engine:** Jekyll (Ruby-based static site generator)
- **Plugin:** `jekyll-paginate` (natively supported on GitHub Pages)
- **No theme gem** ŌĆö all layouts and styles are custom, giving full control

## Layout Hierarchy

```
default.html          ŌåÉ Base shell: <!DOCTYPE>, <head>, <header>, <nav>, <main>, <footer>
  Ōö£ŌöĆŌöĆ home.html       ŌåÉ Extends default. Paginated post list using paginator.posts
  Ōö£ŌöĆŌöĆ post.html       ŌåÉ Extends default. Single article with meta, content, tags
  ŌööŌöĆŌöĆ category.html   ŌåÉ Extends default. Filtered post list by category_name
```

### default.html
- Defines the full HTML document structure
- Includes site-wide `<header>` with nav links (Home, LLM, About)
- Renders `{{ content }}` in `<main>`
- Supports optional `extra_css` frontmatter for page-specific styles
- Links to `/css/style.css`

### home.html
- Iterates over `paginator.posts` (not `site.posts`) for pagination support
- Each post rendered as a `.post-card` with date, category, title link, and truncated excerpt
- Pagination nav (Newer / Older) appears only when `paginator.total_pages > 1`

### post.html
- Displays article metadata: date, author (falls back to `site.author`), category
- Renders full Markdown content inside `.content` div
- Shows tags as `#tag` spans if present

### category.html
- Receives `category_name` and `category_description` from the page's frontmatter
- Filters `site.posts` by matching category, sorted by date descending

## URL Architecture

| Content | URL Pattern | Source |
|---|---|---|
| Homepage | `/` | `index.html` |
| Paginated pages | `/page/2/`, `/page/3/` | Generated by jekyll-paginate |
| Articles | `/articles/<slug>/` | `_posts/` via permalink config |
| Category pages | `/categories/<name>/` | `categories/<name>.md` |
| About | `/about/` | `about.md` via permalink |

## Post Frontmatter Schema

```yaml
---
title: "string"           # Required. Article title
date: YYYY-MM-DD          # Required. Publication date
author: "string"          # Optional. Falls back to site.author
category: "string"        # Required. Must match a category page's category_name
tags: [string, ...]       # Optional. Displayed as #tag badges
excerpt: "string"         # Optional. Used on homepage cards (truncated to 30 words)
---
```

## Deployment

1. Push to `master` branch
2. GitHub Pages automatically builds and deploys
3. Live at `https://janmaru.github.io` within ~1 minute
4. Build status: visible in repo Settings ŌåÆ Pages, or via `gh api repos/janmaru/janmaru.github.io/pages/builds`

## Constraints (GitHub Pages)

- Only [whitelisted plugins](https://pages.github.com/versions/) are supported
- No server-side processing (no search, no dynamic routes)
- `jekyll-paginate` only works on `index.html` at the root
- Custom plugins require switching to GitHub Actions for the build
